## 2. 핵심 아이디어 (Core Logic)
- 재귀함수에 매개변수 2개를 활용한다. 재귀의 깊이를 추적하는 매개변수를 추가로 가져간다.
- 목표하는 값을 기준으로 비교하여, 재귀가 진행 중일 때와 가장 깊은 지점에 도달했을 때, 다른 내용을 출력한다.

## 3. 어려웠던 점 (Difficulties)
- 재귀함수 전후에 코드를 배치해 대칭구조를 구현할 수 있다는 건 알고 있었지만, 이 문제처럼 접두사를 누적하고 base case에서 다른 로직을 처리하는 복잡한 상황에 적용하는 것이 어려웠다. 



---
## 4. 새롭게 배운 점 (What I Learned)

- 재귀 호출 스택을 활용한 대칭 구조 구현
  - 재귀 함수는 호출 스택에 차곡차곡 쌓였다가, 가장 깊은 호출부터 하나씩 실행이 완료되며 스택에서 빠져나온다.
  - 이 원리를 이용해 재귀 호출 전과 후에 코드를 배치하면, 자연스럽게 대칭되는 로직을 구현할 수 있다.


- 재귀함수에 매개변수 2개를 활용해 대칭구조 출력
```python
def func(n, current_depth=1):
    # Base Case: 가장 깊은 지점에 도달하면 재귀를 멈춤
    if current_depth > n:  
        return
    
    # 재귀 호출 전
    print(current_depth, end=' ')

    # 재귀 호출: 더 깊은 지점으로
    func(n, current_depth+1)
    
    # 재귀 호출 후
    if current_depth < n:
        print(current_depth, end=' ')

func(3)  # 1 2 3 2 1 출력
```
