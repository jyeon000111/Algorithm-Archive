## 1. 문제 정보 (Problem)

  - **출처 및 번호**: [백준 2566번]('https://www.acmicpc.net/problem/2566')
  - **난이도**: 브론즈 3
  - **문제 요약**: 
    - 9 * 9 격자판에서 최댓값을 찾아 출력한다.
    - 다음 줄에 몇 행 몇 열에 위치한 수인지도 출력한다.

## 2. 핵심 아이디어 (Core Logic)
- (최댓값, 행 번호, 열 번호) 튜플로 묶어 함께 저장할까?
  - 튜플은 변경 불가능한 객체라, 수정할 때마다 계속 새로 생성되는 비효율
- 변경 가능한 리스트를 사용하는 게 효율적이라고 판단함.
  -> 착각! 지금 코드에선 어차피 재할당이 이루어져서 똑같음.
  - 그리고 재할당이 그렇게 자주 이루어지는 상황도 아니라서 무방함.

## 3. 어려웠던 점 (Difficulties)
- Edge case (반례)
  - [최댓값, 행번호, 열번호] 리스트를 [0, 0, 0]으로 초기화함으로써 생기는 문제!
  - 9*9 격자판이 0으로만 가득할 때, 업데이트가 되지 않음.
  - 최댓값을 -1으로 초기 설정하면 해결됨.


- 착각한 점
  ```python
  my_list = [1, 2, 3]
  my_list = [4, 5, 6]  # 재할당 -> 새로운 메모리 주소에 저장
  ```
  ```python
  my_list = [1, 2, 3]
  my_list[0] = 4
  my_list[1] = 5
  my_list[2] = 6  # 인덱싱: 내용 변경 -> 기존 메모리 주소 유지
  ```
  ```python
  my_list = [1, 2, 3]
  my_list[:] = [4, 5, 6]  # 슬라이싱1: 내용 변경 -> 기존 메모리 주소 유지

  my_list = my_list[:]  # 슬라이싱2: 복사해서 새로운 리스트 생성
  ```

- `변수 = ...` (변수가 통째로 `=` 왼쪽에 있을 때)

- `변수[...] = ...` (`[]`가 변수 이름에 붙어있을 때)
  - 기존 객체의 내용물을 바꾸는 것 (메모리 주소 그대로)
